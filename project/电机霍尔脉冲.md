# Motor Hall Pulse & Angle Acquisition

## Overview

Two independent data paths provide angle/pulse information. Both originate from the same Hall sensor GPIO interrupts but serve different purposes and are accessed via different Modbus registers.

---

## Path 1: Actual Angle (Processed)

**Modbus register:** `0x2731` (REG_REAL_ANGLE, int16, 0.1° unit, read-only)

**Data flow:**

```
Hall A/B GPIO pins (PA9/PA10)
    │  EXTINT interrupt (both edges)
    ▼
motor_hall_instance_t
    │  hall_pulse_counts[0] += 1  (Hall A ISR)
    │  hall_pulse_counts[1] += 1  (Hall B ISR)
    │  direction detection via Hall B falling edge + Hall A level
    ▼
MotorHall_Device_t  (dev_motor_hall.c)
    │  MotorHall_Device_Update() every 1ms
    │  reads motor_hall instance → cached_rpm, cached_direction, total_pulse_count
    │  publishes TOPIC_MOTOR_SPEED_FEEDBACK event
    ▼
RTurn_Device_t  (dev_rturn.c)
    │  subscribes to TOPIC_MOTOR_SPEED_FEEDBACK
    │  subscribes to TOPIC_CURRENT_ALARM
    │  RTurn_UpdateAngle() → integrates angular velocity over time
    │  formula: angle += RPM × 360 / reduction_ratio / 60 × Δt
    │  calibration: overcurrent → re-locate to limit angle
    ▼
g_RealTimeData.real_angle  (int16, 0.1°)
    │  updated via RealTime_UpdateFromDevice()
    │  reads from RTurn device via Device_Control(CMD_RTURN_GET_ANGLE)
    ▼
Modbus read 0x2731 → returns g_RealTimeData.real_angle
```

**Key characteristics:**
- Angle is calibrated against close/open limit positions (-2° and 88°)
- Automatically stops motor at limit angles
- Overcurrent triggers automatic angle re-calibration
- Resolution: 0.1°
- Range: -2° to 88° (configurable via registers 0x271C / 0x271D)

## Path 2: Raw Hall Pulse Accumulator

**Modbus registers:** `0x3714` (low 16 bits) + `0x3715` (high 16 bits) = `g_s32HallPulseAccum` (int32, read-only)

**Reset:** Write `0x0000` to `0x3714` via Modbus function 0x06

**Data flow:**

```
Hall A/B GPIO pins (PA9/PA10)
    │  EXTINT interrupt (both edges)
    ▼
motor_hall_instance_t
    │  hall_pulse_counts[0] += 1  (Hall A ISR, every edge)
    │  hall_pulse_counts[1] += 1  (Hall B ISR, every edge)
    │  (these counters never reset except at motor_hall_reset_counts())
    ▼
motor_hall_update()  (Motor_hall.c, runs every 1ms)
    │  total = hall_pulse_counts[0] + hall_pulse_counts[1]
    │  delta = total - inst->last_total
    │  direction = inst->current_direction
    │      (falls back to inst->last_valid_direction if STOP/NONE)
    │
    │  if direction == FWD:  g_s32HallPulseAccum += delta
    │  if direction == REV:  g_s32HallPulseAccum -= delta
    │
    │  inst->last_total = total   (instance member, reset with counts)
    │  inst->first_run flag handles initial baseline capture
    ▼
g_s32HallPulseAccum  (volatile int32, RAM only, power-on = 0)
    │
    │  Param_ReadByReg(0x3714) → low  16 bits of g_s32HallPulseAccum
    │  Param_ReadByReg(0x3715) → high 16 bits of g_s32HallPulseAccum
    ▼
Modbus read 0x3714 + 0x3715 (2 registers, 0x03)
    → big-endian: [regL_H, regL_L, regH_H, regH_L]
```

**Angle conversion formula:**

```
angle(°) = g_s32HallPulseAccum / 12 / reduction_ratio × 360
```

Where:
- `12` = pulses per motor revolution (3 pole pairs × 2 halls × 2 edges)
- `reduction_ratio` = register 0x3712 value ÷ 10 (default 1183.0, unit 0.1)
- `360` = degrees per output revolution

Example: `3654 / 12 / 1183 × 360 = 92.66°`

**Key characteristics:**
- Raw counter, no calibration or limit-checking
- Positive = CCW (open window), Negative = CW (close window)
- Direction-dependent: FWD adds, REV subtracts
- Accumulator runs every 1ms (not batched)
- Uses `last_valid_direction` fallback to avoid losing pulses when motor stops
- Reset via Modbus write 0 to 0x3714 (only clears `g_s32HallPulseAccum`, NOT the raw `hall_pulse_counts`)

## Direction Detection

Both paths rely on the same direction detection in `hall_b_irq_callback()` (Motor_hall.c):

```
Hall B falling edge detected
    → read Hall A level
    → if (g_AppParam.motor_hall_dir != 0):  // inverted
          A=high → REV, A=low → FWD
      else:                                   // normal
          A=high → FWD, A=low → REV
    → requires 3 consecutive matching detections to confirm direction change
    → requires rpm > 0 (direction detection skipped at zero speed)
```

Motor hall direction can be inverted via Modbus register `0x3710` (REG_MOTOR_HALL_DIR).

---

## Summary

| Feature | Path 1 (Angle) | Path 2 (Pulse Accumulator) |
|---------|---------------|---------------------------|
| Modbus register | 0x2731 | 0x3714 + 0x3715 |
| Type | int16 (0.1°) | int32 (raw pulses) |
| Calibration | Yes (limit-position) | No |
| Reset | Via calibration | Write 0 to 0x3714 |
| Stops at limit | Yes | No (keeps counting) |
| Overcurrent handling | Re-calibrates angle | Unaffected |
| Underlying source | RTurn angle integrator | Hall pulse counter delta |
| Update rate | 1ms (angle integration) | 1ms (delta accumulation) |
